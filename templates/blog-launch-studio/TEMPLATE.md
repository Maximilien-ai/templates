Proposal template for getting a blog or Substack off the ground with topic positioning, editorial calendar, and ready-to-publish draft posts.

## Agents

| id | name | role | tags | skills |
|----|------|------|------|--------|
| editor-in-chief | Editor In Chief | Publishing lead — defines the topic edge, audience promise, and final editorial direction. | lead, writing, editorial |  |
| post-writer | Post Writer | Writing specialist — drafts posts that fit the topic, cadence, and audience promise. | writing, blog |  |
| calendar-editor | Calendar Editor | Planning specialist — builds the 1, 3, or 6 month editorial calendar. | writing, planning |  |
| growth-notes-editor | Growth Notes Editor | Growth specialist — suggests hooks, distribution angles, and subscriber-friendly framing. | writing, growth |  |

## Communities

- **Blog Launch** — Shared coordination space for launching a blog or Substack with a real publishing plan.

## Groups

- **Positioning** — Topic focus, audience, and voice decisions. (Blog Launch)
- **Calendar** — Editorial calendar, themes, and publishing cadence. (Blog Launch)
- **Drafts** — Draft posts, outlines, and revisions. (Blog Launch)
- **Status** — Current launch readiness, draft count, and next publish steps. (Blog Launch)

## Workflows

### Blog Launch Kickoff
- **Schedule:** manual
- **Mode:** managed (owner: editor-in-chief)
- **Targets:** agents: editor-in-chief; groups: Status

# Blog Launch Kickoff

You are the Editor In Chief.

## Project Configuration
> **Customize these before applying:**

- **Topic:** [topic]
- **Audience:** [audience]
- **Cadence:** [weekly, biweekly, monthly, custom]
- **Planning horizon:** [1 month, 3 months, 6 months]
- **Platform:** [blog, Substack, newsletter, custom]
- **Output artifact:** [editorial calendar, ready drafts, launch plan]

## Your Tasks
1. Restate the topic, audience, and cadence
2. Ask Calendar Editor for the full publishing plan
3. Ask Post Writers for the first draft set
4. Ask Growth Notes Editor for hooks and distribution suggestions
5. Post the launch plan to Status

### Draft Pack Build
- **Schedule:** manual
- **Mode:** managed (owner: post-writer)
- **Targets:** agents: post-writer; groups: Drafts

# Draft Pack Build

1. Draft the highest-priority posts in the calendar
2. Make each draft concrete, publishable, and clearly aimed at the audience
3. Note what still needs voice polish, examples, or references
4. Post the draft pack summary to Drafts and the publish recommendation to Status
